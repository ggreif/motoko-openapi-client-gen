## Step 4: Set Up dfx Identity for Mops ✓

`dfx identity new mops` does not work for this. Instead, import the PEM from Bitwarden via stdin:

```bash
# Import PEM from Bitwarden (paste PEM content, then Ctrl-D)
dfx identity import --ic mops /dev/stdin

# Then link the dfx identity to mops
npx ic-mops user import -- "$(dfx identity export mops)"

# Verify
npx ic-mops user get-principal
```

## Step 5: Publish

From `samples/client/spotify/motoko/generated/`:

```bash
cd samples/client/spotify/motoko/generated

# IMPORTANT: GitHub deps must have 40-char commit hashes (not branch/PR refs).
# If mops.toml has bare refs (e.g. #refs/pull/468/head), resolve them first:
npx ic-mops update core
npx ic-mops update serde
npx ic-mops update byte-utils@0.1.2
# This rewrites the URLs with @<40-char-hash> appended.

# Publish — skip docs (files not in src/), benchmarks, and tests (already typechecked)
# Confirm "Publish anyway?" y/y/y for the three GitHub deps, then enter mops identity password.
npx ic-mops publish --no-docs --no-bench --no-test --verbose
```

`--verbose` shows every file being uploaded — confirm `Apis/` and `Models/` files appear.

**What gets uploaded:** All `.mo` files matching `["Apis/**/*.mo", "Models/**/*.mo"]`,
plus `mops.toml` and `README.md`.

**Note:** The template `mops.toml.mustache` now has pinned commit hashes, so freshly generated
packages won't need the `mops update` step.
